# Deployment workflow

This project deploys to Vercel automatically on every push to `main`.

**After completing any feature or fix, always:**
1. Stage all changed/new files with `git add`
2. Commit with a descriptive message
3. Push with `git push`

Do this at the end of every coding session without waiting to be asked. If multiple features were built in a session, a single bundled commit is fine.
